# 最適化手法の比較: CMA-ES vs ウイルス蔓延最適化 (VSO)

2次元ベンチマーク関数を用いて、標準的な進化戦略 **CMA-ES** と、ウイルスの蔓延を模倣した独自手法 **VSO (Virus Spread Optimizer)** を比較する実験です。

---

## ベンチマーク関数

最適化の文脈では、「どれだけ上手く最小値を探せるか」をテストするために人工的な関数を使います。  
ここでは 2次元（変数が x₁, x₂ の2つ）の関数を 6種類用います。

| 関数名 | 特性 | 探索範囲 | 難しさのポイント |
|---|---|---|---|
| **Sphere** | 単純な椀型 | [-5, 5] | 簡単。アルゴリズムの基本動作確認に使う |
| **Rosenbrock** | バナナ型の細長い谷 | [-2, 2] | 谷の形が曲がっており、谷底をたどる必要がある |
| **Rastrigin** | 無数の局所最小値（凹凸が多い） | [-5.12, 5.12] | 途中の窪みに引っかかりやすく、大域探索力が試される |
| **Ackley** | 中央に深い穴、周囲に細かい波 | [-5, 5] | 細かい波に惑わされず中央の最適解に到達できるか |
| **Himmelblau** | 同じ高さの最適解が4箇所 | [-5, 5] | どの最適解に収束しても正解。複数コロニーを維持できるか |
| **Beale** | 複雑な曲面に1つの最適解 | [-4.5, 4.5] | 谷の方向が分かりにくい |

各関数の最適値はすべて **0**（または極めて0に近い値）です。

---

## 手法の説明

### CMA-ES (Covariance Matrix Adaptation Evolution Strategy)

**一言で:** 「良かった候補点の分布を学習しながら、次世代の候補点を生成する進化戦略」

#### 動作の流れ

1. 現在の「中心点」と「探索の広がり（σ）」を元に、複数の候補点をランダムに生成する
2. 各候補点で関数値 f(x) を計算する
3. 良かった点（f値が小さかった点）を中心に、次の探索分布を更新する
   - 平均（中心）が良い方向へ移動する
   - 共分散行列（探索の「形」）が更新され、細長い谷にも追従できるようになる
4. 1〜3 を繰り返す
5. 収束を検出したら停止する

ポイントは「探索の形を自分で学習する」点です。CMA-ES は谷が斜めになっていても、その形に合わせた分布を作り、効率的に最適解へ近づきます。

#### パラメータ

| パラメータ | 値 | 意味 |
|---|---|---|
| `sigma0` | 1.0 | 初期の探索範囲の広さ |
| `max_evals` | 5000 | 関数評価の上限回数 |
| マルチスタート | 無効 | 収束後は停止 |

---

### VSO — Virus Spread Optimizer（ウイルス蔓延最適化）

#### アイデアの出発点

「**関数値が低い場所 = 人口密度が高い場所**」と見なします。  
ウイルスは人の多い場所ほど広まりやすいので、**f(x) が低い場所ほど感染力（子個体の生成確率）が高い**というルールで探索します。

- 個体（感染者）が各地点に存在し、世代ごとに子孫を生む
- 感染力の高い地点（f が低い領域）に個体が自然と集まる
- 一定の「寿命」を過ぎた個体は消滅し、入れ替わりで新しい個体が生まれる

#### 1世代の流れ

```
1. エリート（コロニー中心）の特定
   └─ f値が良く かつ互いに十分離れた個体を最大 n_elite_max 個保護する

2. 消滅判定
   └─ age > lifespan の個体は死亡（エリートは不死）

3. 感染確率の計算
   └─ f値が低い個体ほど高い確率で「親」に選ばれる（softmax変換）

4. 子個体の生成
   ├─ 近傍感染（1 - air_ratio の割合）
   │   └─ 親の位置 + ガウスノイズ  ※親が年老いるほどノイズが小さくなる
   └─ 空気感染（air_ratio の割合）
       └─ 探索空間全体のランダムな場所に出現

5. 死亡スロットに新個体を配置、age = 0 でリセット
```

#### 子個体の生成（近傍感染 vs 空気感染）

| 種類 | 生成方法 | 役割 |
|---|---|---|
| **近傍感染** | 親の位置にガウスノイズを加える | 有望な領域を掘り下げる（局所探索） |
| **空気感染** | 探索範囲内のランダムな点 | 見逃している最適解を発見する（大域探索） |

近傍感染では親の年齢に応じてノイズの大きさが変化します:

```
σ_i = σ_base × σ_decay^(親の年齢)
```

若い個体（age が小さい）→ 広めのノイズ → 周辺を大まかに探索  
老いた個体（age が大きい）→ 小さなノイズ → 最適解付近を精密に掘り下げ

#### エリート・コロニー管理（ニッチ選択）

単純に「f値が最も良い個体だけを保護」すると、最初に見つかった最適解付近にのみ個体が集中してしまいます。Himmelblau 関数のように最適解が複数ある場合に問題になります。

そこで、**空間的に離れた複数の個体を同時に保護**します:

1. f値の良い順に候補を並べる
2. すでに保護している個体と **niche_radius 以上離れている**場合のみ保護に追加する
3. 上限 n_elite_max まで繰り返す
4. 品質が一定水準（現個体群の上位25%以内）に満たない個体は保護しない

これにより、離れた複数の最適解付近に独立したコロニーが自然に形成されます。コロニー数は固定ではなく、良い解が見つかるたびに自動的に増加します。

#### パラメータ一覧

| パラメータ | デフォルト値 | 意味 |
|---|---|---|
| `n_pop` | 20 | 個体数（感染者数） |
| `lifespan` | 5 | 寿命（世代数） |
| `sigma` | 0.2 | 初期の近傍感染半径（探索範囲に対する比率） |
| `sigma_decay` | 0.99 | 世代ごとの感染半径の縮小率 |
| `air_ratio` | 0.2 | 空気感染の割合（新個体のうち何割をランダムに生成するか） |
| `n_elite_max` | 6 | 保護するコロニー中心の最大数 |
| `niche_radius` | 1.0 | コロニー間の最小距離（これ以下なら同じコロニーとみなす） |
| `temperature` | 1.0 | 親選択のランダム性（大きいほど均一、小さいほど貪欲） |
| `stagnation_limit` | 2000 | 改善がなければこの評価回数後に早期停止 |
| `elite_quality_factor` | 1.0 | エリート候補の品質閾値（現集団の上位どの範囲まで許容するか） |

## 実験条件

| 設定 | 値 |
|---|---|
| 試行回数 | 10 run（seed=0, 100, 200, ..., 900） |
| 評価上限 | 5,000 回 |
| 成功判定 | best f ≤ 1e-4 |
| 次元数 | 2次元 |

両手法は同じ評価回数の予算で比較します。CMA-ES は収束後に停止し、VSO は 2,000 回改善がなければ早期終了します。

---

## 結果

10 run の平均値と成功率（best f ≤ 1e-4 の割合）。

| 関数 | カテゴリ | CMA-ES mean | CMA-ES 成功率 | VSO mean | VSO 成功率 |
|---|---|---|---|---|---|
| Sphere | 単峰 | 1.6e-17 | 100% | 1.7e-17 | 100% |
| Rosenbrock | 単峰 | 1.7e-17 | 100% | 2.0e-5 | 100% |
| Rastrigin | 多峰 | 3.6e-1 | 60% | 4.0e-1 | 60% |
| Ackley | 多峰 | 4.3e-12 | 100% | 1.7e-8 | 100% |
| Himmelblau | 複数最適解 | 3.0e-17 | 100% | 3.8e-16 | 100% |
| Beale | 複数最適解 | 2.9e-17 | 100% | 1.0e-12 | 100% |

**考察:**
- **単純な関数（Sphere, Himmelblau）**: 両手法ともほぼ同等の精度で最適解を発見
- **多峰性関数（Rastrigin）**: 両手法とも苦戦。VSO の大域探索能力が一定の優位性を示す場面もある
- **精密収束（Ackley, Beale）**: CMA-ES のほうが最終精度で優位。VSO はニッチ選択により多様な解を維持するため、単一解への収束速度はやや遅い

## 可視化の見方

実行後、`results/{関数名}/` に2種類の図が保存されます。

### trajectory.png — 探索軌跡

```
薄い点   : すべての評価点（どこを探索したか）
折れ線   : best f が更新されたときの位置を時系列で結んだ軌跡
★        : 真の最適解の位置
背景色   : 関数の等高線（暗い = f が低い = 最適解に近い）
```

### convergence.png — 収束曲線

```
x 軸 : 関数の評価回数
y 軸 : その時点までの best f（対数スケール）
線   : 10 run の中央値
影   : 四分位範囲（25%〜75%）
```

両手法を同一グラフに表示するため、x 軸のスケールは全手法で統一されています。

## ファイル構成

```
optimization/
├── benchmarks.py   # ベンチマーク関数の定義（6関数）
├── optimizers.py   # CMAESOptimizer / VirusOptimizer の実装
├── runner.py       # 複数 run の実験実行・統計サマリー
├── visualize.py    # 等高線+軌跡図・収束曲線の生成
├── main.py         # エントリーポイント
└── results/
    └── {関数名}/
        ├── trajectory.png
        └── convergence.png
```

新しい手法を追加する場合は、`optimizers.py` で `BaseOptimizer` を継承したクラスを作成し、`main.py` の `OPTIMIZERS` 辞書に追加するだけで比較実験が動きます。
